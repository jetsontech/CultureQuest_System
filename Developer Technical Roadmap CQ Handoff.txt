Developer Technical Roadmap
To fully compete with Netflix/Tubi, the developer should prioritize the following upgrades:

XMLTV EPG Integration: Modify app/views.py to ingest external XMLTV links (like EPGTalk's US Guide) to provide real-time programming data instead of static loops.

HLS.js Optimization: In LivePlayer.tsx, enable abrEwmaDefaultEstimate and configure capLevelToPlayerSize to allow the player to seamlessly switch between resolutions based on user bandwidth.

UI Rails Logic: Update home.html to group channels by the category field in the database, creating horizontal scrolling rows (e.g., "Trending Movies," "Breaking News").

Logo Management: Implement a Python script to download remote channel logos from M3U metadata and store them in app/static/logos/ to prevent broken image links.


1. Content & Metadata Verification
[ ] Logo Reliability: All remote channel logos should be downloaded and served locally from app/static/logos/. Broken images are the #1 "amateur" look for new platforms.

[ ] EPG Depth: Ensure the generate_schedule.py script is running as a daily cron job to maintain at least 72 hours of forward-looking guide data.

[ ] Content Rights Audit: Confirm that every imported M3U8 link is from a source authorized for rebroadcast (e.g., iptv-org's legal collection).

2. Technical Performance (HLS & Server)
[ ] CORS Policy: Configure your Flask server (or Nginx reverse proxy) to allow cross-origin requests for .m3u8 and .ts files. Without this, external players won't be able to load your streams.

[ ] ABR Check: Verify that the LivePlayer correctly handles Adaptive Bitrate (ABR) switching. Test it by throttling your browser speed to see if it drops from 1080p to 480p without crashing.

[ ] Database Indexing: Ensure the schedules table has an index on starts_at and channel_id. As you add hundreds of channels, un-indexed queries will make the homepage load slowly.

3. Security & Stability
[ ] Stream Header Spoofing: Some high-quality FAST streams require a specific User-Agent (e.g., "Mozilla/5.0"). Ensure your streams route can proxy these headers if necessary.

[ ] Session Security: Change the SECRET_KEY in app/__init__.py from the default 'dev-secret' to a strong, random string before going live.

[ ] Error Handling: Implement a "fallback loop" (a 30-second loop of "Live Content Resuming Soon") if a stream URL returns a 404 or 403 error.

4. UI/UX Final Polish
[ ] Rail Categories: Ensure the "Pluto-style" rails on home.html actually filter by the category field so that "Movies" only shows movie channels.

[ ] Mobile Responsiveness: Test the Beacon grid on a mobile device. The guide rows should stack or become a horizontal scroll to stay readable.

[ ] Favicon & Branding: Replace the default Flask icon with a custom CultureQuest (CQ) favicon.
